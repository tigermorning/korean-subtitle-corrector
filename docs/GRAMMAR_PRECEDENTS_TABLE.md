# 미구현 문법 규칙 사전 검증표

`PRD.md`에서 분리된 문서(구 §18, 선행 조사, 2026-07-17 설계).

`subtitle_corrector/gananda_precedents.py`는 `word_exists()`가 실제로 조회하는 정확한 문자열에만 대응되므로, 아직 코드로 구현되지 않은 문법 규칙(예: `docs/KNOWN_LIMITATIONS.md`의 "제42항 의존명사 잔여 6개": 들/뿐/만/지/차/판)은 그 형식으로 조사할 수 없다 — 코드가 없으니 정확한 쿼리 문자열 자체가 아직 정해지지 않았기 때문이다. 그런데 이런 규칙들은 이미 `docs/KNOWN_LIMITATIONS.md`에 "언젠가 구현할 백로그"로 명시되어 있으므로, 나중에 실제로 코드를 짤 때 온라인가나다를 처음부터 다시 뒤지는 이중 작업을 피하기 위해, `gananda-precedent-research` 배치 조사가 이 백로그 항목도 함께 조사해 아래 표에 미리 정답을 쌓아둔다 — "검증표 먼저, 코드는 나중"(PRD.md §5) 원칙을 코드 작성 이전 단계까지 확장한 것.

**조사 범위는 `docs/KNOWN_LIMITATIONS.md`에 이미 이름이 명시된 백로그 항목으로 한정한다.** 온라인가나다 전체를 무작정 훑는 게 아니라, 이 도구가 언젠가 실제로 쓸 것으로 이미 확정된 문법만 대상으로 삼아 범위가 무한정 커지지 않도록 한다.

규칙이 실제로 코드로 구현되는 시점에는, 이 표의 판정 기준을 그대로 코드에 반영하고, 필요하면 그때 정해진 정확한 쿼리 문자열로 `word_exists()` 스타일 개별 항목을 뽑아 `gananda_precedents.py`로 옮긴다.
