## 1. 띄어쓰기 기본 원칙

- 한국어는 원래 띄어쓰기가 없었고 구한말 선교사들이 도입 — 배경 지식, 규칙 아님
- 체언·용언·수식언(관형사/부사)은 띄어 씀이 원칙, 조사·접두사·접미사·어간·어미는
  붙임 — **[구현됨]** (`_ATTACH_TAGS`/`_MANDATORY_BOUNDARY_TAGS`, 제41항)
- 판별 순서: 사전에 통째로 검색(합성어=하이픈 표기는 무조건 붙임, 명사구=캐럿
  표기는 띔 원칙+붙임 허용) → 접사 여부 확인 → 고유명사/전문용어 여부 확인 →
  그래도 안 되면 못 붙임 — **[구현됨]** (`word_exists`/`compound_status` 우선순위)
- 고유명사(마포 경찰서 강력 팀→마포경찰서 강력팀 허용)·전문용어(만성 골수성
  백혈병→만성골수성백혈병 허용)는 원칙은 띄어쓰기, 붙이려면 전부 붙임(부분만
  붙이는 것 불가) — **[구현됨]** (`_TERM_COMPOUND_TAGS`, §21)
- **사전 검색 실전 팁(연산자)**: `?`는 정확한 글자수 와일드카드(`?꾼`), `*`는
  글자수 무관 와일드카드(`*산맥`); 자세히 찾기에서 "원어" 지정 후 한자 검색도
  가능 — 이건 사람이 표준국어대사전 웹을 검색할 때 쓰는 방법이라 이 엔진의
  `dictionary.py`가 API로 하는 것과 원리는 같음. **[구현됨]**(개념적으로 대응)

## 2. 의존명사·수 표기

- 단위성 의존명사: 순서/차례를 뜻하거나 아라비아 숫자 뒤에 오면 붙임 허용
  (일층/1층, 24명 등) — **[백로그 후보]** — 지금은 "20만" 같은 순수 숫자 표현만
  다룸(제44항), "일층" 류의 순서성 의존명사는 아직 코드 없음
- "제-"(접두사)는 항상 붙임; "차"(의존명사)는 띔 원칙이나 숫자/차례 뒤엔 붙임
  허용("제1차"="제일차"=허용) — **[구현됨]** (`_TERM_COMPOUND_TAGS`의 XPN,
  "차"는 `_DURATION_UNIT_NNB`에 없어 아직 별도 처리는 안 됨 — 백로그 후보)
- "대"(수를 세는 의존명사, "꿀밤 10대")는 숫자와 붙임 가능; "대"(대립 vs., "3
  대 1")는 항상 띔, 예외 "일대일" — **[백로그 후보]**
- 숫자 뒤 한글 단위는 띔, 단위는 한국어보다 영문 선호(3만km 원칙, "3만 km"는
  띄어 쓴 예시로 나오지만 실제로는 "3만km" 형태 선호로 해석됨 — 원문 표현이
  다소 모호해 재확인 필요), 만 단위 이내 붙임(제44항) — **[일부 구현됨]**
  (20만류만 §21에서 처리)
- 아라비아 숫자는 세 자리마다 쉼표, 한글 숫자는 네 자리마다 띄어씀 — **[백로그
  후보]**

변경) — **[구현됨]** (`common_errors.py`의 DISCRIMINATORY_TERMS/PURIFIED_TERMS)

## 4. 국어사전 사용 원칙

- 표준국어대사전 기준, 우리말샘은 보조, 신어사전/오픈사전/영한사전의 한국어
  맞춤법은 신뢰 금지 — **[구현됨]** (`dictionary.py`가 정확히 이 우선순위를
  따름 — stdict 우선, opendict 보조)
- 북한어·방언 사용 금지(예: "가득차다"는 전남 방언, 표준어는 "가득 차다") —
  **[구현됨]**(`word_exists`가 방언 표시가 있는 우리말샘 항목은 표준으로
  인정하지 않는 로직과 원리가 같음 — 다만 "방언" 태그 자체를 명시적으로 거르는
  코드는 아직 없어 완전히 커버되는지는 개별 확인 필요, 백로그 후보)
- 같은 한자, 다른 한글 표기(모음/ㄴ받침 뒤엔 "-율", 그 외 받침 뒤엔 "-률") —
  **[구현됨]**(사전 API 조회로 이미 커버됨, 별도 규칙 불필요)

## 5. 고유명사 띄어쓰기 (지명/거리/음식명)

외국 지명은 붙임, "대로/로/가"는 붙임이지만 "길/거리"는 원칙적으로 띄되 고유명사로
보아 붙임 허용, 요리·음료명은 붙이되(양고기안심스테이크, 아이스모히토) 식재료명은
띔(바닐라 챤탈리 크림), 자연/행정구역/문화 관련 한자어 접미 요소(해/섬/강/산/
산맥/만/반도/호/곶/주/평야/고원, 성/관/도/현/부/시/가, 어/인/족/왕/궁/사/성/요)는
한국어·외래어 구분 없이 항상 붙임 — **[범위 밖]**(§13에서 이미 "고유명사 외래어
표기는 항상 확인 플래그"로 정책 확정, 세부 분류까지는 백로그 후보)

## 6. 외래어 표기법 (외표법 노트 + 구글 문서 전체)

IPA 대조표 기반 세부 규칙(어말 파열음 받침/으 표기, [oʊ]→오, -ton→턴, 어말
s+/z/→스 등 예외 다수), 인명/지명 원지음 원칙(관용 표현 우선 예외 다수: 머라이어
캐리, 마이클 '에어' 조던 등), 중국(1911년 신해혁명 기준 한자어/중국어 구분),
일본(항상 일본어 발음, 한글라이즈 활용), 상표명(한국 지사 등록상표 우선: 폭스바겐,
포르쉐), 형용사형 변환(Persian→페르시아), 언어별 발음기호 확보처(Longman,
Collins, 한글라이즈, Pronounce Names 등), 표기 검색 절차 — **[범위 밖]**
`docs/KNOWN_LIMITATIONS.md`에 이미 명시: "이 규칙 엔진은 영어 하나만 다루는
것보다 훨씬 큰 작업, v2 이후 백로그". 지금 엔진은 kornorms(국립국어원 외래어
용례) API 조회로만 처리.

## 9. 맞춤법 세부 규칙 (이 엔진과 가장 밀접)

- 명사 뒤 접사처럼 쓰이는 경우("오렌지잎", "연잎색")는 명사구 뒤엔 못 붙임 —
  **[백로그 후보]**
- 불규칙 활용형(누렇다→누레/누러니/누렇소) — **[구현됨]**
  (§21에서 `.startswith("VV")`로 불규칙 활용 어간 태그 대응)
- "이따"(단독) vs "있다"(좀 있다 봐), "인제" vs "이제"(둘 다 표준어, 뜻 다름) —
  **[백로그 후보]**
- "부딪치다" vs "부딪히다"(주어의 능동적 움직임 여부) — **이미 [구현됨]**
  (`common_errors.py`의 CONFUSABLE_PAIRS에 이미 있음 — 확인 완료, 추가 작업 불필요)
- 종결어미: "-요/이요"(2021년부터 "빵이요"도 허용), "-(ㄴ)대/-(ㄴ)데", "-재"
  ("그러재"가 맞음, "그러쟤" 아님), "-던"(과거) vs "-든", "-구만"(x)→"-구먼"(o),
  "동사+지 않느냐" vs "형용사+지 않으냐" — **[일부 구현됨]** ("요" 하나는
  §17/§20에서 처리, 나머지 종결어미 구분은 백로그)
- **동사/접사 구분법(시키다/당하다/하다/받다/되다/드리다)** — 이 자료에서 가장
  자세히 다뤄지는 부분, 실제 검증된 정답(퀴즈) 다수 확보:
  - 원칙: 동작성 없는 명사 뒤엔 동사(띄움: 짜장면 시키다, 상 받다, 팀장 되다),
    동작성 있는 명사 뒤엔 접사(붙임: 심부름시키다, 사랑받다, 해체되다),
    의성어/의태어 뒤엔 동사가 원칙이나 사전에 한 단어로 등재되어 있으면 붙임
    (이러쿵저러쿵하다)
  - **예외**: 동작성 없어도 피동 의미가 있으면 접사(상처받다, 스트레스받다) —
    **[구현됨]**(§22, `_is_action_noun`+`_PASSIVE_ONLY_BATDA_NOUNS`, "받다"만)
  - **"-되다"전용 추가 조건**: 동작성 있는 명사라도 전체 단어에 피동성이 없으면
    띔("도움 되는" — 도움은 동작성 있지만 피동 의미 없음, "도움받다"가 피동형)
  - **"-드리다"전용**: 동사 뒤에는 보조동사("보내드리다"), "주다"가 붙어 이미
    합성어가 된 단어는 존대형 "드리다"도 붙임("건네드리다", "도와드리다")
  - **검증된 정답 사례(향후 테스트로 활용 가능)**: "전화받다"(o, 전화는 동작성),
    "카톡 해"(카톡은 동작성 없음), "미움받는"(미움 동작성 있음), "임플란트해야겠다"
    (동작성 있음), "선물받았어"(동작성+피동 둘 다), "배달시켜서"(동작성 있음),
    "조리돌림당했다"(동작성 있음), "도움 되는"(동작성 있으나 피동 아님, 띔),
    "일 시키는"(띔 — "일"은 동작성 없는 일반명사로 처리된 것으로 보임, "일하다"는
    있지만 이 경우 "일"이 구체적 사물 대상처럼 쓰임에 유의), "보이스 피싱 당했어"
    (보이스 피싱은 명사구라 아직 못 붙임 — 명사구는 접사화 대상이 아니라는
    원칙과 일치), "꽝 하는"/"우당탕 하는"(의성어라 띔이 원칙) — **[백로그 후보]**:
    이 검증표를 그대로 `_is_action_noun` 계열 함수의 테스트 케이스로 옮기고,
    시키다/당하다/하다/되다/드리다에도 같은 판별 로직을 확장하면 좋음
- **흔한 오류**: "완전 재밌어"는 틀림("완전"은 명사라 부사로 못 씀), "그리고
  나서/그리고는/그러고서" — **[일부 구현됨]** (그리고 나서/그리고는은 이미
  `ALWAYS_WRONG`에 있음, "완전 재밌어"는 백로그 후보). "요오드→아이오딘"은
  **[구현됨]** (2026-07-17) — 처음엔 표준국어대사전만 조회해 "요오드"도 정상
  표제어로 등재되어 있다고 판단했으나, 우리말샘에 "표준 용어는 '아이오딘'이다"
  라는 비표준 안내가 별도로 붙어 있음을 놓쳤던 것으로 확인됨(사용자 재지적으로
  발견). `_opendict_item_is_standard()`가 "규범 표기는" 문구만 확인하고 전문
  용어에 쓰는 "표준 용어는" 문구를 놓치던 버그, `word_exists()`가 표준국어대사전
  적중 시 우리말샘의 비표준 안내를 확인하지 않고 곧바로 True를 반환하던 버그,
  이렇게 두 가지를 수정하고 `correct_nonstandard_terms()`를 신규 구현해
  자동 교정에 연결함(kornorms는 오히려 "요오드"를 정답으로 등재해 두고 있어
  기존 kornorms 기반 `correct_loanwords`로는 잡을 수 없는 사례였음). 이 과정에서
  "집"(거처, 표준)과 "집"(즙의 옛 표기, 비표준)처럼 같은 표제어를 공유하는
  동형이의어가 있으면 첫 번째 검색 결과만 보고 오판할 위험이 있음도 함께 발견,
  모든 동형이의어 중 하나라도 표준이면 전체를 표준으로 간주하도록 일반화함.
- "대로": 의존명사면 띔(본 대로, 느낀 대로), 조사면 붙임(법대로, 나름대로) —
  **[백로그 후보]**
- "-ㄹ걸"(종결어미, 붙임: 모를걸?) vs "-ㄹ 걸"("것을"로 치환되면 의존명사, 띔:
  먹을 걸 좀 챙겼어) — **[백로그 후보]**
- 의존명사 "뿐"("것뿐"), "나름"(앞에 명사 필요), "만큼"(조사면 붙임, 의존명사면
  띔) — **[백로그 후보]** (제42항 잔여 항목, §5에 이미 "다음 검증 대상"으로 기록)
- "맨"(순純 한국어 접두사, 항상 붙임) — **[백로그 후보]**
- 수효 표기(한번/한 번, 한잔/한 잔, 한입/한 입 — 행위면 붙임, 수효면 띔) —
  **[범위 밖, 의도적]** `docs/IMPLEMENTATION_LOG.md` §13에 "판단 보류, 항상
  확인 필요"로 이미 명시된 대표적 애매 사례
- "뭐 하다"(의문: 너 거기서 뭐 하니?) vs "뭐하다"(특정 용법: 정 뭐하면 하지 마) —
  **[백로그 후보]**
- "안 되다"(금지) vs "안되다"(안타깝다) — **[구현됨]** (§20 `_andoeda_forces_split`)
- **"못 하다"/"못하다"**(안되다/안 되다와 완전히 같은 구조: 못하다=능력부족↔잘하다,
  못 하다=방해·능력전무↔하다), **"잘못하다"/"잘 못하다"/"잘 안 하다"**,
  **"잘 하다"(적절히)/잘되다(성공)/잘 되다(쉽게)**, **"안되다"(낭패)/"안 되다"
  (부정·미달)** — **[백로그 후보, 우선순위 높음]** — "안되다/안 되다"에 이미
  적용한 조건·전제 어미 기반 문맥 판별 방식을 그대로 확장 적용 가능해 보임
- 문법 규칙에 따른 3단계 분리 패턴(관형사/부사 뒤엔 반드시 분리: "뭔 생각 하냐"
  →"생각하다" 분리 불가, "더 어두운 색"→"어두운색" 분리 불가) — **[백로그 후보]**
- **"명사+명사+하다" 3단계 판단** (구체적 예문 다수 확보): 두 명사가 이미
  하나의 합성어/명사구로 등재되어 있는지, 뒤 명사에 "하다"가 접사로 붙을 수
  있는지(동작성)를 함께 봐야 함 — 예: "국어 공부했어"(o, "국어"+"공부"는 별개
  명사구라 "공부"만 "하다"와 결합), "초과 근무했다"(o, "초과^근무"가 명사구),
  "통행금지하는"(o, "통행금지"가 동작성 명사로 한 단어처럼 붙음), "상호 협력했다"
  (o, "상호"가 부사라서 "협력하다"와만 결합) — **[백로그 후보, 흥미로움]**:
  이미 있는 `_compound_candidate_spans()`(합성어 판정)와 `_is_action_noun()`
  (동작성 판정)을 조합하면 상당 부분 커버될 것으로 보임
- "N 같다"("처럼"으로 치환되면 붙임: 꿈같은 추억), "N+만 하다"(친구만 한 게
  없다) — **[백로그 후보]**
- 개별 관용구 예외 목록: 볼일, 거참, 처하다/쳐주다, 흰모래/흰옷, 한 번만 봐줘,
  데려다줘, 일주일 치, 신경 쓰다, 가득 차다, 다시 한번 — **[백로그 후보]**
  (일부는 이미 사전 조회로 커버될 가능성 높음, 개별 검증 필요)
- 의미가 달라지는 띄어쓰기(집어던지다/집어 던지다, 뱃속/집안/다음날/뭐하다/
  못쓰다/잘살다-잘 살다 등 관용구성 복합어) — **[일부 구현됨]** (뱃속/입안류
  맞춤법 개정 사항은 §5의 compound_status 사전 조회로 자동 커버됨)

## 10. 번역 스타일/표현

영문법 번역 규칙, 번역투 지양(영어식 피동 "-되어지다", 사역 "-게 하다", 이중
피동/사동, 사물 주어 지양, 한정사/빈도부사 생략, 직접인용→간접인용 등), 구어체/
문어체 구분, 한국인 말습관 반영, 슬랭/관용구/게임 용어 번역, 분야별 전문용어
(건축/의료/경찰/군대) — **[범위 밖]** `docs/IMPLEMENTATION_LOG.md` §13에 이미
명시: "번역투|오역|윤문
섹션은 아예 성격이 다른 카테고리 — 규정상 맞다/틀리다가 아니라 번역 스타일
판단이라 이 도구의 핵심 원칙(권위 있는 규범 근거 기반)으로 검증할 방법이
없다." 전문용어 사전도 §9에 "도메인별 전문용어 사전"으로 이미 v2 이후
백로그로 분류됨.

## 11. 체크리스트 (자가 점검용)

시키다/당하다/하다/받다/되다/드리다, 뱃속/집안/다음날, 외래어 표기(알콜→알코올,
나트륨→소듐), 자막 나이 표기(말자막 "-살" vs 화면자막 "-세"), 단위 범위 표기
("6만-8만 명"), "&"→및/겸/또는, "넣을지 말지 헷갈리면 빼라"(의/것/게 되다/게
하다 등 군더더기 표현 지양) — 대부분 위 항목들과 중복되거나 **[범위 밖]**
(번역 스타일)에 해당.

## 정리: 이 문서에서 나온 새 백로그 후보 (우선순위 순, 2026-07-17 갱신)

1. **"시키다/당하다/하다/되다/드리다"에도 "받다"와 같은 동작성/피동성 판별 로직
   확장** — 이미 검증된 정답표(§9 "검증된 정답 사례" 참고)까지 확보되어 있어
   착수하기 가장 쉬움
2. **"명사+명사+하다" 3단계 판단** — 기존 `_compound_candidate_spans`/
   `_is_action_noun`을 조합하면 될 것으로 보이는 흥미로운 확장
3. **"못 하다"/"못하다", "잘못하다"류** — "안되다/안 되다"와 동일 구조라 재사용 가능
4. 순서성 의존명사(제-/층/대/차) 확장, 의존명사 "뿐"/"나름"/"만큼", "맨" 접두사
5. "-ㄹ걸"/"-ㄹ 걸", "대로"(조사/의존명사), "뭐 하다"/"뭐하다"
6. 흔한 오류 추가("완전 재밌어" 등)를 `ALWAYS_WRONG`에 반영 ("요오드→아이오딘"은
   `correct_nonstandard_terms()`로 2026-07-17 구현 완료, 위 §9 참고)
